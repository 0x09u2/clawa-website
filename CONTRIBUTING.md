# Contributing to Clawa

Thank you for your interest in contributing to **Clawa**.  
Clawa is an open-source, community-governed ecosystem focused on modular blockchain infrastructure and decentralized governance.

We welcome contributions of all kinds.

---

## Ways to Contribute

You can contribute by:

- Improving documentation
- Fixing bugs or refactoring code
- Proposing new features or modules
- Submitting Clawa Improvement Proposals (CIPs)
- Participating in discussions and governance

---

## Getting Started

1. Fork the repository
2. Create a new branch from `main`
3. Make your changes
4. Ensure your code is clear and well-documented
5. Open a Pull Request with a concise description

---

## Code Standards

- Keep changes minimal and focused
- Follow existing project structure and style
- Write clear commit messages
- Avoid introducing breaking changes without discussion

---

## Governance & Proposals

Larger changes, new modules, or governance-related updates should be proposed via a **Clawa Improvement Proposal (CIP)** before implementation.

CIPs allow the community to review, discuss, and align on decisions.

---

## Community Standards

All contributors must follow the project's **Code of Conduct**.  
Respectful, inclusive, and constructive collaboration is expected at all times.

---

## Questions or Ideas?

If you're unsure where to start, open an issue or join the community discussion channels.
