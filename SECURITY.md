# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability related to Clawa, please report it
responsibly.

Do **not** open a public issue for security-sensitive reports.

Instead, contact the maintainers privately via:
- GitHub Security Advisories (preferred), or
- Official communication channels listed on the Clawa website

---

## Scope

This security policy applies to:

- Website code
- Tooling and infrastructure maintained in this repository
- Governance-related software maintained by the Clawa project

---

## Response Process

Reported vulnerabilities will be reviewed as soon as possible.

If confirmed, fixes will be developed and released in a timely manner,
followed by public disclosure when appropriate.

---

## Responsible Disclosure

We appreciate responsible disclosure and collaboration to improve
the security of the Clawa ecosystem.
